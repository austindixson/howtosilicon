Daily driver · 32 GB Apple Silicon

Qwen3.6 35B-A3B-4bit + Ferroclaw context engine ON

Serve a long-horizon coding agent on one Mac with ~32 GB unified memory. Weights (~19 GB) leave ~7–8 GB for KV. The engine keeps usable input under ~8k tokens by externalizing tool bulk to disk and packing history turn-safely.

Measured

  • Multiturn tool-bulk prompt: 6 889 tokens (~23 s wall)
  • Same seed engine OFF: 35 250 tokens (~129 s) → 5.12× / 5.7×
  • Mid-file fact recovery from capsules: 100%
  • Peak residency (typical): approach ~21 GB before pressure scale / quarantine

Stack

  • Model: mlx-community/Qwen3.6-35B-A3B-4bit
  • Server: python -m hermes_context_memory.local_qwen on :8080
  • Harness: Ferroclaw [context_memory] profile = "mac32"
  • Tools: unlimited lifetime (max_tool_calls_total = 0)

Config skeleton

[agent]
default_model = "mlx-community/Qwen3.6-35B-A3B-4bit"
max_iterations = 500
token_budget = 500000
max_tool_calls_per_iteration = 12
max_tool_calls_total = 0

[providers.openai]
base_url = "http://127.0.0.1:8080/v1"
api_key_env = "OPENAI_API_KEY"
max_tokens = 4096

[context_memory]
enabled = true
profile = "mac32"
hard_local_budget = true
safe_context_tokens = 10240
safe_input_tokens = 7680
target_input_tokens = 6400
output_reserve_tokens = 1024
tool_reserve_tokens = 1024
externalize_tool_results_over_tokens = 800

Start

# Terminal A — MLX
export PYTHONPATH=~/Desktop/hermes-context-memory-main
~/venvs/mlx/bin/python -m hermes_context_memory.local_qwen \
  --model ~/models/huggingface/hub/models--mlx-community--Qwen3.6-35B-A3B-4bit/snapshots/... \
  --python ~/venvs/mlx/bin/python --port 8080

# Terminal B — agent
export OPENAI_API_KEY=local
export FERROCLAW_HEADLESS=1   # optional
ferroclaw exec "your long task"

Why ON is the daily driver

Context is a cache. Disk holds evidence capsules and task state. Progressive completion steers finish audits without PARTIAL / tool-budget tombstones. Engine OFF is a control arm only.
