Long-horizon repo audit (orca-class)

Open-ended senior-dev audit of a large tree without PARTIAL tombstones and without tool-count circuit breakers.

Runtime laws

  • max_tool_calls_total = 0 — unlimited lifetime tools
  • Emergency pack + continue (never pack + quit)
  • Progressive completion steers: nudge → pack → optional text-only finish
  • Accept only multi-section reports (reject PARTIAL / tool-budget stubs)
  • Discovery-first: list / glob before inventing paths

Command

export OPENAI_API_KEY=local
export FERROCLAW_HEADLESS=1

ferroclaw exec "Senior-dev audit of /path/to/repo.

Discovery: list_dir/glob first; only read listed paths.
Deliver COMPLETE report (not PARTIAL): overview, architecture,
risks, ≥5 findings with real paths, recommendations."

Proven outcome

Headless audit of orca-coder-main produced a full senior-dev report (~12k chars): architecture, risks, five findings, recommendations — with no tool budget reached and no PARTIAL title. See ferroclaw docs/reports/orca-coder-audit.md.

What we rejected

  • Soft-finish PARTIAL artifacts when tools or iterations hit a counter
  • Stripping tools to force synthesize early
  • Empty ERROR on CM over-budget
